## SQL Lessons

This section documents my structured learning and hands-on practice in SQL, with a focus on building a strong foundation in data querying, transformation, and relational database design.

---

### Project Structure

**Section 1: Database Setup & Query Foundations**

- **`01_queryclauses_database.sql`**  
  Defines a practice database (`mydatabase`) including schema creation, table definitions, and sample data.  
  This environment is used to explore and practice core SQL query concepts.

- **`02_sales_database.sql`**  
  Contains a more comprehensive database (`salesdb`) with multiple related tables, including customers, employees, products, orders, and orders_archive.  
  This dataset simulates real-world business scenarios and supports more advanced analytical queries.

- **`03_query_clauses.sql`**  
  Focuses on practical SQL querying using the `mydatabase` schema.  
  Covers data retrieval, filtering, sorting, and aggregation techniques used in day-to-day data analysis.


### Learning Approach

This work follows a structured, step-by-step approach designed to mirror real-world data workflows:

1. **Database Design & Setup**
   - Created databases and schemas.  
   - Defined tables with appropriate structure.  
   - Inserted sample data to simulate real datasets.  

2. **Query Development**
   - Practiced writing SQL queries to extract and analyze data.  
   - Focused on understanding how different SQL clauses interact.  
   - Built queries that progressively combine multiple operations.  


### SQL Concepts Covered

The following core SQL concepts have been implemented and practiced:

- **SELECT** – retrieving complete and partial datasets.  
- **WHERE** – filtering data based on conditions.  
- **ORDER BY** – sorting query results.  
- **GROUP BY** – aggregating data. 
- **HAVING** – filtering aggregated results.  
- **DISTINCT** – removing duplicate values.  
- **LIMIT / TOP** – controlling result set size.  
- **Query Combination** – integrating multiple clauses into a single query.  

**Additional Techniques:**
- Using static values within queries.  
- Applying conditional logic with **CASE statements**.  
- Executing multiple queries within a script.  


### Key Outcomes

- Developed a strong foundation in SQL query structure and logic.  
- Gained practical experience in filtering, sorting, and aggregating data. 
- Learned how to transform raw data into meaningful insights.  
- Built confidence working with relational databases and structured datasets.
